# Gold CNN-BiLSTM Local Simulation Bundle

This package was built from your uploaded files so you can run the **gold new-data simulation locally** for the requested test window:

- **Simulation start:** 2025-05-01
- **Simulation end:** 2025-11-26

## What is included

- the **gold PyTorch CNN-BiLSTM model artifacts** for all uploaded `gold_RRL_interpolate_6am` seeds
- the **historical gold dataset** you uploaded, saved locally as:
  - `data/historical/gold_RRL_interpolate_history_to_2025_04_30.csv`
- a **local simulation script**:
  - `scripts/run_simulation.py`
- a **prepared notebook copy** and the **original uploaded notebook**
- `requirements.txt` and `environment.yml`
- Windows and shell helper launchers
- a **template CSV** for the required new-data file:
  - `data/new/gold_RRL_interpolate_2025_05_01_to_2025_11_26_TEMPLATE.csv`
- a **required session-date file** for the requested window:
  - `data/new/required_comex_sessions_2025_05_01_to_2025_11_26.csv`

## Important limitation

Your uploaded materials did **not** include the actual prepared **new-data CSV** covering **2025-05-01 to 2025-11-26**.  
The uploaded historical file ends at **2025-04-30**.

Because of that, this bundle includes the full runnable environment and simulation pipeline, but the exact file below still needs to be filled with the actual aligned test-period data:

- `data/new/gold_RRL_interpolate_2025_05_01_to_2025_11_26.csv`

## Required columns for the new-data CSV

The simulation expects these columns exactly:

1. `Date`
2. `Gold_Futures`
3. `Silver_Futures`
4. `Crude_Oil_Futures`
5. `UST10Y_Treasury_Yield`
6. `Federal_Funds_Rate`
7. `Employment_Pop_Ratio`
8. `gepu`
9. `gpr_daily`

## Local directory paths

Use these local paths inside the extracted folder:

- historical data: `data/historical/gold_RRL_interpolate_history_to_2025_04_30.csv`
- new test data: `data/new/gold_RRL_interpolate_2025_05_01_to_2025_11_26.csv`
- template for new test data: `data/new/gold_RRL_interpolate_2025_05_01_to_2025_11_26_TEMPLATE.csv`
- model root: `models/gold_RRL_interpolate_6am/`
- default seed folder used by the uploaded notebook: `models/gold_RRL_interpolate_6am/seed_2/`
- outputs: `outputs/`

## Quick start on Windows

### Option A

1. Run `setup_env.bat`
2. Put your prepared new-data CSV here:
   - `data/new/gold_RRL_interpolate_2025_05_01_to_2025_11_26.csv`
3. Run `run_simulation.bat`

### Option B

Run manually:

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python scripts\run_simulation.py
```

## Quick start on macOS / Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python scripts/run_simulation.py
```

## Default simulation command

```bash
python scripts/run_simulation.py \
  --seed 2 \
  --history-csv data/historical/gold_RRL_interpolate_history_to_2025_04_30.csv \
  --new-data-csv data/new/gold_RRL_interpolate_2025_05_01_to_2025_11_26.csv \
  --sim-start 2025-05-01 \
  --sim-end 2025-11-26 \
  --query-date 2025-08-15
```

## Outputs generated by the script

The script saves these files into `outputs/`:

- `gold_simulation_all_results_seed_<seed>.csv`
- `gold_simulation_window_2025-05-01_to_2025-11-26_seed_<seed>.csv`
- `gold_simulation_metrics_2025-05-01_to_2025-11-26_seed_<seed>.json`
- `gold_simulation_plot_2025-05-01_to_2025-11-26_seed_<seed>.png`
- `gold_lookup_<query-date>_seed_<seed>.csv` if the queried date exists

## Notes

- The default seed is **2** because your uploaded simulation notebook was pointed to `seed_2`.
- You can switch seeds with `--seed 0`, `--seed 1`, `--seed 2`, `--seed 42`, `--seed 99`, or `--seed 123`.
- The session template file contains **150 COMEX session dates** from **2025-05-01** to **2025-11-26**.

## Flask GUI / Live Simulation API

A complete Flask dashboard has been added to this bundle.

Run it with:

Windows:

```bat
setup_env.bat
run_flask_gui.bat
```

Mac/Linux:

```bash
bash setup_env.sh
bash run_flask_gui.sh
```

Then open:

```text
http://127.0.0.1:5000
```

The browser dashboard lets you enter a today/anchor date and click **Next Day Predict** to reveal one forecast row at a time. It updates predicted price, actual price, RMSE, R², and the moving graph after every click.

Read the detailed Flask instructions in:

```text
README_FLASK_GUI.md
```
